U - UNDERSTAND (2 min): Restate problem, ask constraints

M - MATCH (1 min): Identify pattern

P - PLAN (3-5 min): Brute force FIRST, then optimize

I - IMPLEMENT (15-25 min): Think aloud, descriptive names

R - REVIEW (3-5 min): Test edge cases

E - EVALUATE (2 min): State time/space complexity

EDGE CASES CHECKLIST

□ Empty input
□ Single element
□ All same elements
□ Negative numbers/zero
□ Maximum constraints
□ Duplicates

COMMON BUGS

□ Off-by-one (i < n vs i <= n)
□ Array out of bounds
□ Forgot to increment loop
□ Wrong comparison operator
□ Mutating while iterating

TS GOTCHAS

⚠️ arr.sort() sorts as strings!
⚠️ shift()/unshift() is O(n)
⚠️ Math.max(...huge) crashes
⚠️ Always use === not ==
