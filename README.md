## üéØ How Interactive Sessions Work

### Phase 1: Video Assignment (5 min)
**You:** "Claude, start session 1 1"
**Me:** "Starting Day 1, Session 1: Big O & Arrays. Watch this 20-min video: [link]. Say 'I watched the video' when ready."

### Phase 2: Concept Check (5-10 min)
**You:** "Claude, I watched the video"
**Me:** "Great! Let's verify understanding. Explain Big O notation in your own words."
[I quiz you on 3-5 concepts]

### Phase 3: Tips & Tricks (5 min)
**You:** "Claude, give me tips"
**Me:** "Here are 3 essential tips: [teaches non-obvious insights]"

### Phase 4: Problem Solving (30-40 min per problem)
**You:** "Claude, give me the problem" or "Go"
**Me:** "Here's your problem: Two Sum. [reads problem]. What clarifying questions do you have?"

[You follow UMPIRE method while I observe]
- Ask clarifying questions (I answer as interviewer)
- Explain approach (I ask probing questions)
- Code solution (I stay silent, take notes)
- Test solution (I point out missed cases)

**You:** "Claude, I'm done coding"
**Me:** "Walk me through your solution with an example."

### Phase 5: Evaluation (5 min)
**You:** "Claude, evaluate me"
**Me:** [Structured feedback using rubric]

### Phase 6: Next Problem or Session
**You:** "Claude, next problem" - Continues session
**You:** "Claude, next session" - Auto-commits, moves to next session
